Atelier 3

Pour cette dernniere étape nous allons utilisé un notifier et un proxy. On part du principe que les mécanismes de filtres seront effectué coté serveur via une requête ajax et le tout en utilisant des notifications.

Classe Controller

Le controller demande filtrage via une notification et sera informé du résultat pour mettre à jour la vue via une autre notification.

  • pour lancer le filtre, il utilise le notifier en lançant la notification FILTER
  • pour le résultat, il s'abonne via le notifier à la notification FILTER_OK

Classe Notifier

Cette classe permet de gérer les notifications globales. Elle propose une fonction d'abonnement (register) et de notification (notify)

Classe Proxy

Cette classe , en réponse à une notification FILTER, va effectuer un appel ajax vers le serveur pour obtenir les résultats filtrés. A la réception elle notifie l'application par une notification FILTER_OK
